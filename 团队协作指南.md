## 快速接入（3分钟上手）

### 第一步：配置 Git（仅命令行用户需要）

```bash
# 设置用户名（替换为你的名字）
git config --global user.name "你的名字"

# 设置邮箱
git config --global user.email "你的邮箱@example.com"

# 查看配置
git config --list
```

### 第二步：克隆仓库

#### 在 Cursor 中克隆
1. 打开 Cursor
2. 点击左侧 **Source Control**（或按 `Ctrl+Shift+P` 输入 Clone）
3. 输入仓库地址：`https://github.com/smooth-QAQ/taidi_cup_B.git`
4. 选择保存位置

#### 在 VSCode 中克隆
1. 打开 VSCode
2. `Ctrl+Shift+P` → 输入 `Git: Clone`
3. 输入仓库地址：`https://github.com/smooth-QAQ/taidi_cup_B.git`

#### 命令行克隆
```bash
# 进入工作目录
cd c:\cursor_Xiangmu

# 克隆仓库
git clone https://github.com/smooth-QAQ/taidi_cup_B.git

# 进入项目目录
cd taidi_cup_B
```

> **注意**：如果提示需要登录，输入你的 GitHub 用户名和密码（或 Personal Access Token）

### 第三步：创建功能分支

#### 在 Cursor/VSCode 中创建
1. 点击左下角 **main** 分支图标
2. 选择 **Create new branch**
3. 输入分支名，例如：`feature/lyc-登录模块`

#### 命令行创建
```bash
# 从 main 创建你的功能分支（替换为你的名字和功能）
git checkout -b feature/你的名字-功能描述

# 例如：
git checkout -b feature/lyc-登录模块
git checkout -b feature/zhang-数据处理
```

## Cursor/VSCode 图形化操作

### 查看更改

1. 点击左侧 **Source Control** 图标（或按 `Ctrl+Shift+G`）
2. 可以看到所有已修改的文件
3. 点击文件名可以查看具体更改（diff）

### 提交代码

1. 在 Source Control 面板中
2. 点击文件旁边的 **+** 添加到暂存区（或点击 **Stage All** 暂存所有）
3. 在输入框中输入提交信息，例如：`feat: 添加用户登录功能`
4. 点击 **Commit**（或按 `Ctrl+Enter`）

### 推送代码

1. 提交后，点击 Source Control 面板顶部的 **Publish Changes**
2. 首次推送会自动创建远程分支

### 拉取最新代码

1. 点击左下角的 **main** 分支图标
2. 选择 **Pull, Push** → **Pull from**
3. 选择 **origin/main**

### 创建 Pull Request

1. 推送代码后，VSCode 会提示 **Create Pull Request**
2. 点击后填写 PR 信息
3. 或直接在 GitHub 网页上操作

### 切换分支

1. 点击左下角分支图标
2. 选择 **Switch to Branch...**
3. 输入分支名或从列表中选择

### 解决冲突

1. 当 pull 时出现冲突，文件会显示红色标记
2. 打开冲突文件，会看到类似这样的标记：
   ```
   <<<<<<< HEAD
   你的代码
   =======
   别人的代码
   >>>>>>> feature/xxx
   ```
3. 删除标记，保留需要的代码
4. 保存文件
5. 点击 **Stage Resolution** 确认解决
6. 提交并推送

### 查看提交历史

1. `Ctrl+Shift+P` → 输入 **View History**
2. 或安装 **GitLens** 插件，右键文件查看历史

## 每日开发流程

### 早晨：同步最新代码

#### Cursor/VSCode 操作
1. 点击左下角 **main** 分支
2. **Pull, Push** → **Pull from** → **origin/main**
3. 创建/切换到你的功能分支

#### 命令行操作
```bash
# 确保在 main 分支
git checkout main

# 拉取最新代码
git pull origin main

# 创建/切换到你的功能分支
git checkout feature/你的名字-功能描述
```

### 开发中：提交代码

#### Cursor/VSCode 操作
1. `Ctrl+S` 保存文件
2. Source Control → 查看更改
3. 点击 **+** 暂存文件
4. 输入提交信息
5. 点击 **Commit**

#### 命令行操作
```bash
# 1. 查看当前更改
git status

# 2. 添加要提交的文件
git add 文件名        # 添加单个文件
git add .             # 添加所有更改

# 3. 提交（写清楚做了什么）
git commit -m "feat: 添加用户登录功能"

# 4. 推送到远程
git push origin feature/你的名字-功能描述
```

### 完成功能后：创建 Pull Request

1. 在 GitHub 页面点击 **Compare & pull request**
2. 选择分支：`feature/xxx` → `main`
3. 填写信息后点击 **Create pull request**
4. 等待管理员审核

## 分支命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 功能开发 | `feature/名字-功能` | `feature/lyc-登录模块` |
| 紧急修复 | `hotfix/名字-问题` | `hotfix/lyc-修复登录bug` |
| 文档更新 | `docs/名字-内容` | `docs/lyc-更新README` |

> **规则**：名字用拼音首字母或英文，功能用中文简述，用 `-` 分隔

## 提交信息规范

### 格式

```
<类型>: <简短描述>

[可选的详细说明]
```

### 类型标识

| 标识 | 含义 | 示例 |
|------|------|------|
| `feat:` | 新功能 | `feat: 添加用户登录功能` |
| `fix:` | 修复bug | `fix: 修复登录超时问题` |
| `docs:` | 文档更新 | `docs: 更新README` |
| `refactor:` | 重构代码 | `refactor: 简化数据库查询` |
| `test:` | 测试相关 | `test: 添加单元测试` |
| `chore:` | 其他杂项 | `chore: 更新依赖版本` |

### 示例

```bash
git commit -m "feat: 添加用户注册功能"
git commit -m "fix: 修复数据库连接池泄漏"
git commit -m "docs: 更新操作指南"
```

## Pull Request 流程

### 创建 PR

1. **推送代码后**，GitHub 会显示 **Compare & pull request** 按钮，点击它
2. **填写 PR 信息**：
   - **Title**：`[feat] 用户登录模块`
   - **Description**：
     ```
     ## 做了什么
     - 添加了用户登录页面
     - 集成了后端API
     
     ## 测试情况
     - 本地测试通过
     ```
3. 点击 **Create pull request**

### 等待审核

- 管理员会收到通知
- 请等待审核，**不要**自行合并到 `main`
- 如果需要修改，管理员会留言

### 合并后

- 删除你的功能分支（可选）
- 继续下一个功能开发

## 反悔指南（重要！）

### 场景 1：提交了但还没 push，想取消

#### Cursor/VSCode 操作
1. 打开 **Source Control** 面板
2. 点击 **Undo Last Commit**

#### 命令行操作
```bash
# 撤销最后一次提交，保留更改在本地
git reset --soft HEAD~1

# 或者完全丢弃这次提交
git reset --hard HEAD~1
```

### 场景 2：已 push，想撤回

#### 命令行操作
```bash
# 方法1：撤销提交（推荐，保留历史）
git revert HEAD

# 方法2：强制回退（慎用！改变历史）
git reset --hard HEAD~1
git push --force
```

### 场景 3：想回到之前的某个提交

#### Cursor/VSCode 操作
1. `Ctrl+Shift+P` → 输入 **View Git History**
2. 右键点击目标提交
3. 选择 **Create Branch at This Commit**

#### 命令行操作
```bash
# 查看所有操作历史
git reflog

# 恢复到指定版本
git checkout <commit-id>
```

### 场景 4：文件改错了，想恢复

#### Cursor/VSCode 操作
1. 在 Source Control 面板中右键点击文件
2. 选择 **Discard Changes**

#### 命令行操作
```bash
# 恢复单个文件
git restore filename.py

# 恢复所有文件
git restore .
```

## 常见问题

### Q1：push 时提示 "rejected"

#### Cursor/VSCode 操作
1. 点击左下角分支图标
2. **Pull, Push** → **Pull from** → **origin/main**
3. 如果有冲突，手动解决后重新提交推送

#### 命令行操作
```bash
# 先拉取远程最新代码
git pull origin main

# 如果有冲突，手动解决后重新提交
git add .
git commit -m "解决冲突"
git push
```

### Q2：冲突了怎么办？

#### Cursor/VSCode 操作
1. 打开有冲突的文件
2. 会看到类似这样的标记：
   ```
   <<<<<<< HEAD
   你的代码
   =======
   别人的代码
   >>>>>>> feature/xxx
   ```
3. 点击文件上方的 **Accept Incoming** / **Accept Current** / **Accept Both**
4. 或手动编辑，保留需要的代码
5. 删除标记
6. Source Control → 点击 **Stage Resolution**
7. 提交并推送

### Q3：不小心删除了本地分支

```bash
# 从远程恢复
git checkout -b feature/xxx origin/feature/xxx
```

### Q4：想看某个文件的历史

#### Cursor/VSCode 操作
1. 右键点击文件
2. 选择 **GitLens: View File History**
3. 查看所有修改记录

#### 命令行操作
```bash
git log --oneline filename.py
git show <commit-id>:filename.py
```

## 命令速查表

### 基础操作

```bash
git status              # 查看当前状态
git diff                # 查看更改内容
git log --oneline       # 查看提交历史（简洁）
```

### 分支操作

```bash
git branch              # 查看本地分支
git branch -a           # 查看所有分支（包括远程）
git checkout 分支名     # 切换分支
git checkout -b 分支名  # 创建并切换到新分支
git branch -d 分支名    # 删除本地分支
```

### 同步操作

```bash
git pull                # 拉取并合并
git fetch               # 只拉取不合并
git push                # 推送
git push -u origin 分支  # 首次推送（设置上游分支）
```

### 撤销操作

```bash
git reset HEAD~1        # 撤销本地提交
git revert HEAD         # 创建撤销提交
git restore 文件名       # 恢复单个文件
git restore .           # 恢复所有文件
```
