## 🎯 项目简介

本系统是一个基于LLM的**智能财报问答平台**，能够：

- ✅ **Text2SQL**：将自然语言问题自动转换为SQL查询
- ✅ **智能对话**：多轮对话理解财务问题并给出答案
- ✅ **推理调试**：可视化AI的思考过程
- ✅ **图表生成**：自动生成数据可视化图表

### 核心技术栈

- **LLM**：支持 OpenAI GPT、Gemini、DeepSeek、本地vLLM
- **数据库**：PostgreSQL（结构化数据）+ ChromaDB（向量检索）
- **前端**：Streamlit Web界面
- **Embedding**：OpenAI text-embedding-3-small 或本地BGE模型

## ⚡ 快速开始（3步跑通附件4）

### 前提条件

- [ ] **Python 3.10+** 已安装（推荐 3.10 或 3.11）
- [ ] **Docker Desktop** 已安装并运行（Windows/Mac）或 Docker Engine（Linux）
- [ ] 拥有一个LLM API密钥（推荐 DeepSeek 或 OpenAI）

> **💡 重要提示**：
> - Windows 用户：确保 Docker Desktop 正在运行（任务栏有鲸鱼图标）
> - Mac 用户：安装 Docker Desktop for Mac
> - Linux 用户：安装 Docker 和 Docker Compose

### 步骤1：安装依赖（5分钟）

```bash
# 进入项目目录
cd c:\cursor_Xiangmu\taidi_cup_B\financial-assistant\chatbot_financial_statement

# 创建虚拟环境（二选一）
conda create -y -n text2sql python=3.10
conda activate text2sql

# 或使用 venv
python -m venv venv
venv\Scripts\activate  # Windows
# 或 source venv/bin/activate  # Mac/Linux

# 安装 Python 依赖（使用国内镜像加速）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 如果安装慢，先升级 pip
python -m pip install --upgrade pip
```

> **⚠️ 常见问题**：
> - 如果报错 `pip 不是内部或外部命令` → 虚拟环境未激活成功
> - 如果某个包安装失败 → 换镜像源或单独安装 `pip install package_name`
> - 如果提示 Python 版本不对 → 确认 `python --version` 是 3.10+

### 步骤2：配置环境变量（2分钟）

在 `chatbot_financial_statement/` 目录创建 `.env` 文件：

```ini
# ========== LLM配置（只需填这两行） ==========
OPENAI_API_KEY=sk-your-api-key-here
LLM_HOST=https://api.deepseek.com

# ========== 数据库配置（Docker会自动创建，保持默认） ==========
DB_NAME=financial_db
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432

MONGO_DB_HOST=localhost
MONGO_DB_USER=admin
MONGO_DB_PASSWORD=admin
MONGO_DB_PORT=27017

# ========== Embedding配置（使用OpenAI最简单） ==========
LOCAL_EMBEDDING=False
EMBEDDING_SERVER_URL=https://api.openai.com/v1
EMBEDDING_MODEL=text-embedding-3-small

# Reranker（可选，保持默认）
RERANKER_SERVER_URL=http://localhost:8081
RERANKER_MODEL=BAAI/bge-reranker-v2-m3
```

**API密钥获取**：
- **DeepSeek**（���荐）：https://platform.deepseek.com/ → API Keys → 创建密钥（便宜、速度快，新用户免费额度）
- **OpenAI**：https://platform.openai.com/ → API Keys
- **其他兼容API**：任何 OpenAI 格式的 API 服务均可（如 OneAPI、Together AI 等）

> **⚠️ 重要**：
> 1. `.env` 文件必须在 `chatbot_financial_statement/` 目录下（与 `home.py` 同级）
> 2. API Key 不要有空格或引号
> 3. Windows 用户注意：不要将文件保存为 `.env.txt`（需显示文件扩展名）

### 步骤3：启动系统（5分钟）

```bash
# 1. 启动数据库容器
docker-compose up -d

# 等待 5 秒让容器启动
sleep 5  # Mac/Linux
# Windows 用： timeout /t 5

# 2. 验证容器状态
docker ps
# 应看到 postgres2 和 mongodb2 两个容器正在运行

# 3. 初始化数据（首次运行，耗时 5-10 分钟）
python setup.py --preprocess v3.2 --force True --openai True --vectordb chromadb

# 4. 测试环境
python test.py

# 5. 启动 Web 界面
streamlit run home.py
```

✅ **浏览器自动打开** `http://localhost:8501`，开始使用！

## 🔧 详细配置说明

### 配置选项对比

| 配置项 | 使用 OpenAI Embedding（推荐） | 使用本地 Embedding（需要 GPU） |
|--------|-----------------------------|-----------------------------|
| **LOCAL_EMBEDDING** | `False` | `True` |
| **EMBEDDING_SERVER_URL** | `https://api.openai.com/v1` | `http://localhost:8080` |
| **是否需要 GPU** | ❌ 否 | ✅ 是 |
| **启动速度** | 快（API 调用） | 慢（需下载模型） |
| **成本** | API 调用计费 | 本地免费 |
| **初始化命令** | `--openai True` | `--local True` |

### LLM 选择指南

| LLM 提供商 | 适用场景 | 配置示例 |
|-----------|---------|---------|
| **DeepSeek** | 日常使用（性价比最高） | `LLM_HOST=https://api.deepseek.com` |
| **OpenAI GPT** | 高质量回答 | `LLM_HOST=https://api.openai.com/v1` |
| **Gemini** | Google 生态 | 使用 `GEMINI_API_KEY` |
| **本地 vLLM** | 隐私优先、无 API 费用 | `LLM_HOST=http://localhost:8000/v1` |

### 数据库版本选择

`--preprocess` 参数说明：

| 版本 | 公司数量 | 说明 | 推荐场景 |
|------|---------|------|---------|
| `v3` | 100 家 | 基础版，快速测试 | 快速验证 |
| `v3.2` | 200+ 家 | 完整版，含 QoQ 比率 | **推荐** |
| `v2` | 较少 | 旧版本 | 不推荐 |

## 📊 运行附件4任务

附件4只有一个问题：**"茅台2022年的营业收入是多少？"**

### 方法1：通过 Web 界面（推荐新手）

1. 启动系统后，浏览器访问 `http://localhost:8501`
2. 点击 **"Text2SQL"** 页面
3. 在输入框输入：`茅台2022年的营业收入`
4. 点击发送，查看生成的 SQL 和结果

### 方法2：批量运行脚本（自动化）

```bash
# 在项目根目录执行
python batch_runner.py \
  --input_file 附件4.xlsx \
  --output result_2.xlsx \
  --generate_chart
```

**生成结果**：
- `result_2.xlsx` - 包含答案的 Excel 文件
- `charts/` 目录 - 自动生成的数据可视化图表

### 方法3：直接调用 API

```python
from chatbot_financial_statement.agent.text2sql import Text2SQL
from chatbot_financial_statement.agent.const import Text2SQLConfig

# 初始化
config = Text2SQLConfig(
    llm_api_key="your-api-key",
    llm_base_url="https://api.deepseek.com"
)
t2s = Text2SQL(config)

# 查询
result = t2s.query("茅台2022年的营业收入是多少？")
print(result)
```

## 🔧 常见问题

### Q1：Docker 容器启动失败

**现象**：`docker ps` 看不到 `postgres2` 和 `mongodb2` 容器

**解决**：
```bash
# 1. 确认 Docker Desktop 正在运行
# Windows：任务栏右键 Docker 图标 → 查看状态
# 如果未运行，启动 Docker Desktop

# 2. 查看错误日志
docker-compose logs

# 3. 重启容器
docker-compose down
docker-compose up -d

# 4. 检查端口占用
netstat -an | findstr "5432"  # Windows
# 如果端口被占用，修改 docker-compose.yml 中的端口映射
```

### Q2：数据库连接失败

**现象**：`python test.py` 显示数据库连接错误

**解决**：
```bash
# 1. 检查容器是否运行
docker ps

# 2. 查看容器日志
docker logs postgres2
docker logs mongodb2

# 3. 确认环境变量正确
# 检查 .env 文件中的 DB_HOST、DB_PORT 等配置

# 4. 测试连接
python -c "import psycopg2; conn = psycopg2.connect(host='localhost', port=5432, dbname='financial_db', user='postgres', password='postgres'); print('✅ 连接成功'); conn.close()"
```

### Q3：Embedding API 调用失败

**现象**：报错 `Embedding API error` 或 `401 Unauthorized`

**解决**：

**方案 A：改用 OpenAI Embedding（推荐）**
1. 在 `.env` 中设置：
   ```ini
   LOCAL_EMBEDDING=False
   EMBEDDING_SERVER_URL=https://api.openai.com/v1
   EMBEDDING_MODEL=text-embedding-3-small
   ```
2. 确保 `OPENAI_API_KEY` 有效

**方案 B：使用本地 Embedding（需要 NVIDIA GPU）**
1. 启动本地服务：
   ```bash
   # GPU 版本
   docker-compose --profile local-embedding up -d
   ```
2. 在 `.env` 中设置：
   ```ini
   LOCAL_EMBEDDING=True
   EMBEDDING_SERVER_URL=http://localhost:8080
   ```

### Q4：Python 依赖安装慢/失败

**解决**：使用国内镜像源
```bash
# 临时使用
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 永久配置（推荐）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

### Q5：首次运行 setup.py 很慢

**原因**：需要下载和处理大量财报数据（200+ 家公司）

**解决**：
- 耐心等待 5-10 分钟（首次）
- 后续运行使用 `--force False` 跳过重建
- 或使用小数据版本 `--preprocess v3`（100 家公司）

### Q6：查询结果不准确

**可能原因**：
1. **数据中没有该公司**：检查 `csv/df_company_info.csv` 中是否有目标公司
2. **时间范围不对**：明确指定年份和季度
3. **问题表述不清**：使用更具体的财务术语

**优化建议**：
- 在 Chat 页面中多尝试几次
- 使用 Text2SQL 页面验证 SQL 生成
- 查看 Reasoning 页面了解 AI 的思考过程

## 🆘 获取帮助

### 查看运行状态

```bash
# 查看 Docker 容器
docker ps

# 查看容器日志
docker logs postgres2
docker logs mongodb2

# 查看 Python 进程
# Windows:
tasklist | findstr python
# Mac/Linux:
ps aux | grep python
```

### 重启系统

```bash
# 完整重启
cd chatbot_financial_statement

# 1. 停止容器
docker-compose down

# 2. 重启容器
docker-compose up -d

# 3. 重启 Streamlit（在浏览器中停止后重新运行）
streamlit run home.py
```

### 清理重建

```bash

rm -rf chatbot_financial_statement/ETL/chroma_db  # 删除向量库
rm -rf __pycache__  # 删除 Python 缓存

# 重新开始
docker-compose up -d
python setup.py --preprocess v3.2 --force True --openai True
```
